系统架构

抖音直播 SDK
    ↓ 礼物事件 / 评论事件
Node.js 服务器 (server.js)
    ↓ WebSocket 实时推送
浏览器游戏界面 (index.html)

礼物事件接入

// 在你的抖音 SDK 回调中调用：
fetch('http://your-server:3000/api/gift', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    viewerId:    user.openId,      // 观众唯一ID（必填）
    viewerName:  user.nickname,   // 昵称（可选）
    viewerAvatar: user.avatar,    // 头像URL（可选）
    giftName:    gift.giftName,   // 礼物名称（必填，需在配置表中存在）
    giftCount:   gift.count       // 数量（可选，默认1）
  })
});

评论坐标指令接入

// 在你的评论回调中调用：
fetch('http://your-server:3000/api/comment', {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    viewerId:   user.openId,
    viewerName: user.nickname,
    comment:    comment.text   // 系统自动解析坐标
  })
});

// 支持的坐标格式（观众在直播间评论区输入）：
// "x:300 y:450"   ← 推荐格式
// "x=300,y=450"
// "炮:300,450"
// "300,450"

API 接口一览

方法路径说明
POST/api/gift触发礼物事件（创建炮台）
POST/api/comment发送评论（解析坐标指令）
GET/api/scores获取排行榜（聚合）
POST/api/scores提交积分记录
GET/api/config获取当前配置
POST/api/config保存配置
GET/api/admin/scores获取详细积分记录
DELETE/api/admin/scores清空所有积分

游戏规则说明

  • 观众刷礼物 → 屏幕随机位置出现对应等级的炮台，显示观众头像和昵称
  • 观众在直播间评论区输入坐标（如 "x:300 y:400"）→ 炮台瞄准并立即开炮
  • 倒计时结束前未输入坐标 → 系统自动瞄准最近的鱼开炮
  • 命中范围内的鱼全部捕获，积分 = 鱼的基础分 × 炮弹等级倍率
  • 积分保存30天，自动清理过期数据
